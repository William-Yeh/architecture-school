Ansible 實戰講堂

掌握自動化組態管理最佳實務




課程目標

從實作中正確掌握「自動化組態管理」的整體操作及最佳實務，擁有進一步將 Ansible 及相關工具運用在自身工作場域的能力。

課程簡介

  • DevOps 簡介
  • 自動化組態管理觀念
  • 實習環境介紹
  • Ansible 基本工具
  • Ansible playbook (劇本檔) 實作
  • 虛擬機及雲端環境運用
  • 重構：模組化、一般化
  • Ansible Galaxy
  • 組態測試
  • 部署管理
  • 綜合運用

課程時數

6 小時。

適合對象

本課程適合對 DevOps 自動化感興趣的 IT 從業人員，尤以後端系統為主。因為八成內容會以「實機操作」形式進行，適合不排斥在 Linux 命令列輸入指令的人。

常見問題


► Ansible 和 Chef/Puppet/Salt 有什麼不同？
大體而言，可以從三方面來探討：程式語言、Pull vs Push、ecosystem。
► 這次課程，會提到 Ansible 與 Chef/Puppet/Salt 的差別嗎？
會介紹一點點。

Disclaimer：我的回答可能會過於主觀，而且各家都在截長補短，我並沒有積極追蹤其他三者的最新狀況。 ^_^
► Ansible 能支援 Windows 嗎？
Ansible 1.7 之後，managed host 可以是 Windows 了；但 control machine 仍然需要 Unix 系統。詳情請見官方文件。
► 這次課程，會教 Windows 嗎？
不會。
